# 前端版本号安全读取与显示

## 核心原则

**版本号是构建时注入的，不是运行时 fetch 的。**

---

## 正确做法

### 单一真相来源（Single Source of Truth）

版本号统一存储在项目配置文件（如 `src/config/version.ts`）：

```typescript
// src/config/version.ts
export const APP_VERSION = '26.1.14';
```

组件通过 ES Module 导入读取，**零网络开销**：

```typescript
import { APP_VERSION } from '@/config/version';
```

### 构建时自动注入

构建脚本（如 `scripts/get-version.mjs`）在 `npm run build` 时自动写入版本号，来源优先级：

1. `VERCEL_GIT_TAG`（CI/CD 环境变量）
2. `GITHUB_REF_NAME`（GitHub Actions）
3. `git describe`（本地构建 fallback）
4. `v0.0-dev`（兜底默认值）

### 版本号不显示正确值的根因分析

| 浏览器显示 | 代码值 | 根因 |
|-----------|--------|------|
| `0.0-dev` | 正确（如 `26.1.14`） | **部署问题**：代码未推送 / CDN 缓存未刷新 / 重新部署未完成 |
| 旧版本号 | 正确 | **CDN 缓存问题**：需强制刷新（Cmd+Shift+R）或调整缓存策略 |
| 空白或 undefined | 代码问题 | 导入路径错误 / 构建时未正确替换 |

**结论**：若 `src/config/version.ts` 中版本号值正确，则显示 0.0-dev 是**部署问题，非代码问题**。

---

## 错误做法（反面教材）

- ❌ 前端 `fetch('/version.json')` 获取版本号（多余、慢、CDN 缓存问题）
- ❌ 前端 `fetch('/api/version')`（同上前缀）
- ❌ 运行时通过 `window.__APP_CONFIG__` 读取（构建产物，非源码）

---

## 调试步骤

1. 确认 `src/config/version.ts` 中的 `APP_VERSION` 值是否正确
2. 确认构建日志中版本注入脚本的输出是否正确
3. 若仍显示 0.0-dev：执行 `git push` + 等待 CI/CD 完成 + 强制刷新浏览器
