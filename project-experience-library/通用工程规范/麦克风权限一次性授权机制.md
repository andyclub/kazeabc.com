# 麦克风权限一次性授权机制

## 概述

Web 应用中使用 `navigator.mediaDevices.getUserMedia({ audio: true })` 调用麦克风时，浏览器原生维护的权限持久化机制使用户只需授权一次，后续使用无需重复请求。

---

## 授权逻辑

| 调用时机 | 行为 |
|---------|------|
| 首次调用 | 浏览器弹出麦克风授权请求 |
| 后续调用（同源） | 直接返回 MediaStream，不弹窗 |
| 授权状态 | 按 Origin（域名）跨会话持久化 |

---

## 代码示例

```typescript
// 标准 Web API 调用，无需额外权限处理逻辑
const stream = await navigator.mediaDevices.getUserMedia({ audio: true });
```

---

## 何时需要重新授权

| 情况 | 是否重新授权 |
|------|-------------|
| 用户在浏览器设置中手动撤销 | ✅ 是 |
| 使用不同的浏览器 | ✅ 是 |
| 网站从 HTTP 切换到 HTTPS | ✅ 是 |
| 清除浏览器数据 / 使用隐身模式 | ✅ 是 |
| 用户更换麦克风设备 | ⚠️ 可能 |

---

## 技术原理

```
浏览器权限存储（按 Origin）
──────────────────────────
kazeabc.com → 麦克风: 已授权 ✓
              （跨会话持久化）
```

**关键点**：
- 权限由浏览器管理，不由应用程序控制
- 这是浏览器标准行为，非应用自定义实现
- 应用层只需调用 API，浏览器自动处理后续授权判断

---

## 用户引导文案

> 首次使用时需授权麦克风，之后使用无需重复授权。如需撤回授权，请在浏览器设置中管理网站权限。
